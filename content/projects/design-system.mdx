## 왜 만들었나요?

매번 새 프로젝트를 시작할 때마다 같은 컴포넌트를 반복해서 만드는 것에 지쳤습니다.
일관된 디자인 언어와 재사용 가능한 컴포넌트가 필요했죠.

## 핵심 원칙

1. **접근성 우선**: 모든 컴포넌트는 WCAG 2.1 AA 기준 충족
2. **합성 가능**: 작은 단위로 조합하여 복잡한 UI 구성
3. **커스터마이징**: CSS 변수를 통한 쉬운 테마 변경

## 컴포넌트 목록

### 기본 컴포넌트
- Button, Input, Select, Checkbox, Radio
- Dialog, Drawer, Popover, Tooltip
- Card, Badge, Avatar, Skeleton

### 레이아웃
- Container, Stack, Grid, Flex
- Sidebar, Header, Footer

## 진행 상황

현재 기본 컴포넌트 개발을 완료하고, 복합 패턴과 문서화 작업을 진행 중입니다.
